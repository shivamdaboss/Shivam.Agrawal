Mechanical Projects

Programming

    I programmed the robot with the help of one other teammate. We had a very specific file structure. In addition to the files opcontrol.c, auto.c, and init.c, the three source code files, we used a robot.c and a robot.h function. In robot.c and robot.h, we declared all of the basic variables and functions used by the robot. For example, we had a drive function, lift function, and variables for the buttons. The added functions and variables were to go into their own seperate files. In opcontrol.c, we only called functions.

     Two of the add-on files were for the lcd and were called lcd.c and lcd.h. In these files, all of the lcd functions and variables were declared and defined. For the lcd, we used a struct called slide to define the general features of a slide. We then made an array of the slide structs, and used a counter to cycle through them. In order to recieve values from the buttons, we checked if the button were pressed before but not currently. This allowed for us to do commands on release of the button. We also put a boolean in the slide struct called “pressed” and made it true when a slide was selected. This would allow the slides to stay on their respective screens after a slide was selected. Some of the slides on lcd include motor testing, battary voltage display as well as an auton recorder.

     Another set of files was the auton recorder file set, autonRecorder.c and autonRecorder.h. This file set included all of the declarations and defenitions for functions and variables related to the auton recorder. This program allowed the driver to drive the robot for a specific time based off of the mode selected and the robot would record it. Then the robot could play it back. This program has three steps essentially. The first step is to record. Based off the mode selected, the program will record the robots movements for however many milliseconds. There is a struct called robot with all of the movement values of the robots. Every millisecond, the values of the robot struct are updated and then recorded on a file. The next step is to save the file. There were ten slots that allowed for a 15 second run, and one slot for a 60 second run. The last step is to play it back. In this step, the robot simply reads the values off of the file, and updates the robot struct. Everytime the robot struct is updated, which is every millisecond, functions designed for the playback will take the robot struct values and use them so that the robot will act on them.
